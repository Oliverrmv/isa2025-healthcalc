# isa2025-healthcalc
Health calculator used in Ingeniería del Software Avanzada
Historias de Usuario (User Stories)
Las historias de usuario sirven para definir las funcionalidades o características que un usuario espera encontrar en una aplicación o proyecto. Su enfoque está en los objetivos o necesidades del usuario, proporcionando una visión clara de lo que se busca lograr.

En este proyecto, hemos definido dos historias de usuario, cada una asociada a un método inicial de nuestra calculadora (idealweight y basalmetabolicrate). Para cada historia de usuario, hemos establecido dos criterios de aceptación, que son condiciones o requisitos que deben cumplirse para considerar la historia como completa y operativa.

Estructura de las Historias de Usuario: As a - I want - So that
Para formular una historia de usuario, se sigue la estructura:

As a → Se define el tipo de usuario.
I want → Se expresa la necesidad o requerimiento del usuario.
So that → Se describe el propósito o beneficio que se busca obtener.
Estructura de los Criterios de Aceptación: Given - When - Then
Para definir los criterios de aceptación, se emplea la estructura clásica:

Given → Se establecen las condiciones iniciales o precondiciones.
When → Se indican las acciones que se van a realizar.
Then → Se detalla el resultado esperado tras la ejecución de las acciones.
Cucumber y Gherkin
Siguiendo el patrón Given-When-Then, una de las herramientas más utilizadas es Cucumber, que se basa en el lenguaje Gherkin. Su uso es común en la automatización de pruebas y en metodologías ágiles de desarrollo. Gherkin permite describir el comportamiento esperado de la aplicación o sistema, facilitando la generación de casos de prueba automatizados mediante Cucumber.
